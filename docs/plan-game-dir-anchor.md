# GameSaver 待开发计划：`%GAME_DIR%` 动态锚点

## 背景

当前规则路径已支持 `%USERPROFILE%` 归一化与运行时展开，可覆盖跨用户名迁移。  
但对于“存档位于游戏安装目录”的场景（老游戏、独立游戏、绿色版）仍有明显痛点：

- 机器 A 与机器 B 的游戏安装路径通常不同（盘符/目录结构不同）。
- 导入后规则可存在，但路径不可直接用，导致备份/恢复不可用或误导用户。

目标是引入 `%GAME_DIR%` 动态锚点，降低跨机迁移成本并提升可用性。

---

## 本期目标（MVP）

在不重做学习/导入主流程的前提下，交付以下能力：

1. 支持将游戏目录下的规则路径模板化为 `%GAME_DIR%\\...`。
2. 运行时可基于已绑定 EXE 将 `%GAME_DIR%` 展开为本机真实路径。
3. 当 `%GAME_DIR%` 无法展开时，在预检查中给出明确“需绑定 EXE”提示并阻止关键操作。
4. 兼容旧数据与旧备份槽位命名，避免升级后历史备份不可恢复。

---

## 非目标（本期不做）

- 不实现完整“导入前预览向导（manifest 预检 + 手动映射流程）”。
- 不新增复杂规则编辑 DSL，仅支持固定 token。
- 不做驱动/注入路线改造。

---

## 关键设计决策

### 1) Token 规范

- 新增统一 token：`%GAME_DIR%`
- 与现有 `%USERPROFILE%` 机制并存。
- 禁止并存多套同义 token（如 `<GAME_DIR>`）以避免协议分裂。

### 2) 模板化触发条件（学习/规则写入）

仅当候选路径满足以下条件时才模板化为 `%GAME_DIR%`：

- 路径位于 `exe.parent()` 之下（大小写不敏感前缀匹配）。
- 模板化后仍为有效子路径（不允许 `..` 逃逸）。

其余路径按现有逻辑处理（优先 `%USERPROFILE%`，否则保留绝对路径）。

### 3) 运行时展开优先级

`launch 时传入 exe` > `游戏库绑定 exe` > `无法展开（返回明确错误）`

应用点：

- 备份读写路径
- 回滚恢复路径
- 预检查路径可写性
- 其他依赖 `confirmedPaths` 的运行时 I/O

### 4) 安全与路径约束

- 展开后必须是绝对路径。
- 拒绝包含非法跳转片段（如 `..` 越界）。
- 不允许空路径、根路径退化等异常输入。

### 5) 旧备份兼容策略

备份槽位读取恢复时同时支持：

- 新槽位命名（基于 `%GAME_DIR%` 归一化后的路径）
- 旧槽位命名（历史绝对路径命名）

避免升级后出现“版本存在但恢复找不到槽位”。

## 分阶段实现清单

- [ ] 新增 `%GAME_DIR%` 常量与识别逻辑。
- [ ] 扩展 `normalize_confirmed_path_for_storage`：

- [ ] 扩展 `expand_confirmed_path_for_runtime`：
  - [ ] 支持 `%GAME_DIR%` 展开。
  - [ ] 无可用 EXE 时返回可诊断错误（而非静默）。
- [ ] 增加路径安全校验（非法相对跳转拦截）。

## Phase 2：运行链路接入（后端）

- [ ] 备份流程接入 `%GAME_DIR%` 展开。
- [ ] 恢复流程接入 `%GAME_DIR%` 展开。
- [ ] 启动前检查（precheck）补充 `%GAME_DIR%` 可解析状态。
- [ ] 错误消息统一：
  - [ ] “未绑定 EXE，无法解析 `%GAME_DIR%`”

## Phase 3：前端状态与交互

- [ ] 游戏库卡片增加黄灯提示：需要绑定 EXE 才能解析 `%GAME_DIR%`。

- [ ] 备份版本读取兼容旧槽位命名。
- [ ] 编写一次性迁移说明（文档）。

---

## 风险与应对

### 风险 A：误模板化导致路径错误

- 应对：仅在“明确位于 exe.parent()”且安全校验通过时模板化。

### 风险 B：Program Files 写入失败

- 应对：恢复前执行可写性检查；错误提示改为可操作文案（建议管理员权限）。

### 风险 C：游戏目录噪音误判（log/cache）

- 应对：学习评分继续强化噪音降权；必要时追加更严格扩展名策略。

### 风险 D：升级后历史备份不可恢复

- 应对：恢复路径查找同时支持新旧槽位命名。

---

## 测试矩阵（最小）

- [ ] 同机：`%GAME_DIR%` 规则备份/恢复可用。
- [ ] 跨用户名：`%USERPROFILE%` 与 `%GAME_DIR%` 混合规则正常。
- [ ] 跨盘符：导入后仅绑定一次 EXE 即恢复可用。
- [ ] 未绑定 EXE：预检查黄灯，阻止恢复并给出明确提示。
- [ ] 旧规则升级：历史绝对路径规则仍可用。
- [ ] 旧备份回滚：新版本可识别旧槽位并恢复。

---

## 验收标准（DoD）

- 用户导入迁移包后，如规则含 `%GAME_DIR%`，只需“绑定本地 EXE 一次”即可恢复备份与回滚能力。
- 未绑定 EXE 时不会误执行备份/恢复，且前端有清晰指引。
- `cargo check` 与前端类型检查通过，学习/规则管理/游戏库主流程无回归。
